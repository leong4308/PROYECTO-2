SERVIDOR WEB.

Definición.

Un servidor web es un programa informático que procesa una aplicación del lado del servidor, realizando conexiones bidireccionales y/o unidireccionales y síncronas o asíncronas con el cliente y generando o cediendo una respuesta en cualquier lenguaje o aplicación del lado.

Función.
  • Espera peticiones de clientes (generalmente navegadores web) en el puerto 80.
  • Recibe una petición.
  • Busca el recurso solicitado en su sistema de archivos.
  • Envía el recurso solicitado al cliente.

Tipos principales.

Apache HTTP Server, Apache HTTP Server, LiteSpeed Web Server.etc

Protocolo HTTP.

HTTP es el protocolo de comunicación que permite las transferencias de información en la World Wide Web. Es un protocolo de la capa de aplicación para la transmisión de documentos hipermedia, como HTML.

Características principales:
Sin estado: Cada petición HTTP es independiente y no guarda información de peticiones anteriores.
Cliente-servidor:
El cliente (generalmente un navegador web) inicia la comunicación haciendo unapetición al servidor.
Métodos de petición:
Los más comunes son GET (solicitar un recurso), POST (enviar datos alservidor), PUT (actualizar un recurso), DELETE (eliminar un recurso)
Códigos de estado:
Indican el resultado de la petición (por ejemplo, 200 OK, 404 Not Found, 500 Internal Server Error).
Cabeceras:
Sin estado: Cada petición HTTP es independiente y no guarda información de peticiones anteriores.

Versiones.

HTTP/1: La versión más utilizada, introduce mejoras como conexiones persistentes.

HTTP/2:Introduce multiplexación de flujos, compresión de cabeceras y priorización de peticiones.

HTTP/3: La versión más reciente, basada en el protocolo QUIC para mejorar el rendimiento.

Seguridad.

Configuración adecuada:Asegurar que la configuración del servidor sea segura, deshabilitando funciones innecesarias y siguiendo las mejores prácticas.

Control de acceso:Implementar autenticación fuerte y autorización adecuada para proteger recursos sensibles.

Monitoreo y registro:Implementar sistemas de monitoreo y registro para detectar y responder rápidamente a incidentesde seguridad .

Seguridad en la capa de red:Configurar correctamente firewalls y utilizar técnicas como la segmentación de red.

Implementación de HTTPS:Utilizar SSL/TLS para cifrar la comunicación entre el cliente y el servidor, protegiendo contra ataques de intermediarios

Actualizaciones regulares:Mantener el software del servidor y sus componentes actualizados para corregir vulnerabilidades conocidas

Prevención de ataques comunes:Proteger contra ataques como inyección SQL, cross-site scripting (XSS), y cross-site reques forgery (CSRF) .

Firewalls de aplicaciones web (WAF):Utilizar WAFs para detectar y bloquear ataques maliciosos.

Escalabilidad Horizontal.

Seguridad en la capa de red:

Agregar más servidores para distribuir la carga, utilizando técnicas como el balanceo de carga.

Escalabilidad Vertical:

Aumentar los recursos (CPU, RAM, almacenamiento) de los servidores existentes.

Arquitectura de Microservicios:

Dividir la aplicación en servicios independientes que pueden escalar de forma individual.

Bases de Datos Distribuidas:

Utilizar sistemas de bases de datos que pueden escalar horizontalmente.

Caché Distribuido:

Implementar sistemas de caché distribuidos para reducir la carga en los servidores y bases de datos.
